"เอกสารประกอบการพัฒนาแอปพลิเคชัน ""D-Point"" (Peer Recognition Platform)
1. ภาพรวมและวัตถุประสงค์ (Overview & Objective)

D-Point คือแอปพลิเคชันสำหรับสร้างวัฒนธรรมองค์กรเชิงบวก (Positive Corporate Culture) ผ่านระบบการให้รางวัลและคำชื่นชมระหว่างเพื่อนพนักงาน (Peer-to-Peer Recognition)

วัตถุประสงค์หลัก:

ส่งเสริมการชื่นชมและยอมรับซึ่งกันและกันในที่ทำงาน

สร้างแรงจูงใจและเพิ่มความผูกพันของพนักงานต่อองค์กร (Employee Engagement)

เป็นเครื่องมือวัดผลและเชิดชูพนักงานที่มีผลงานโดดเด่นผ่านการยอมรับของเพื่อนร่วมงาน

2. ผู้ใช้งานระบบ (User Roles)

ระบบจะประกอบด้วยผู้ใช้งาน 2 ประเภทหลัก:

พนักงาน (Employee): ผู้ใช้งานทั่วไป มีสิทธิ์ในการให้และรับเหรียญ, แลกของรางวัล

ผู้ดูแลระบบ (Admin): ผู้ควบคุมระบบทั้งหมด สามารถตั้งค่า, ดูข้อมูลภาพรวม และจัดการส่วนต่างๆ ของแอปพลิเคชัน

3. คุณสมบัติของระบบ (System Features & Requirements)

3.1 ระบบการให้และส่งเหรียญ (Coin Giving System)

การจัดสรรเหรียญ:

พนักงานทุกคนจะได้รับเหรียญสำหรับ ""ส่งต่อ"" จำนวนจำกัด (เช่น 20 เหรียญ) เข้าสู่บัญชีทุกวันที่ 1 ของเดือน

อายุของเหรียญ:

เหรียญสำหรับส่งต่อจะมีอายุ 30 วัน หากพนักงานไม่ส่งเหรียญให้ใครภายในสิ้นเดือน เหรียญที่เหลือจะถูกรีเซ็ตเป็น 0 และจะได้รับโควต้าใหม่ในเดือนถัดไป (ไม่สามารถสะสมโควต้าเหรียญสำหรับส่งต่อได้)

ฟังก์ชันการส่งเหรียญ:

พนักงานสามารถเลือกเพื่อนพนักงานที่ต้องการส่งเหรียญให้ กี่คนก็ได้ตามจำนวนเหรียญที่เหลืออยู่ โดยต้องกรอกรหัสพนักงานที่ต้องการส่งให้

ตัวเลือกการเปิดเผยตัวตน: ผู้ส่งสามารถเลือกว่าจะส่งแบบ ""เปิดเผยชื่อ"" หรือ ""ไม่เปิดเผยชื่อ""

การแนบข้อความ: ผู้ส่งสามารถพิมพ์ข้อความชื่นชมสั้นๆ แนบไปกับเหรียญได้

ระบบกรองคำหยาบ: มีระบบ AI ตรวจจับและบล็อกข้อความที่มีคำไม่สุภาพ เพื่อรักษาบรรยากาศที่ดีในการใช้งาน

3.2 กระเป๋าเก็บเหรียญและระบบสะสมแต้ม (Wallet & Point System)

กระเป๋า (Wallet):

พนักงานทุกคนจะมี ""กระเป๋า"" ส่วนตัวสำหรับเก็บเหรียญที่ ""ได้รับ"" จากผู้อื่น

เหรียญที่ได้รับจะถูกเรียกว่า ""Point"" (แต้ม) และจะไม่มีวันหมดอายุ

การสะสมแต้ม:

พนักงานสามารถสะสม Point ที่ได้รับไปเรื่อยๆ เพื่อใช้แลกของรางวัล

หน้า Wallet จะต้องแสดงจำนวน Point คงเหลือ และประวัติการรับเหรียญ (ได้รับจากใคร, เมื่อไหร่, ข้อความอะไร)

3.3 ระบบแลกของรางวัล (Reward Redemption System)

แคตตาล็อกของรางวัล:

มีหน้าสำหรับแสดงรายการของรางวัลที่พนักงานสามารถใช้ Point แลกได้

ของรางวัลแต่ละชิ้นจะมีการกำหนดจำนวน Point ที่ต้องใช้ในการแลก

กระบวนการแลก:

พนักงานสามารถเลือกของรางวัลและกดยืนยันการแลก ระบบจะตัด Point ออกจาก Wallet โดยอัตโนมัติ

มีระบบแจ้งเตือน (Notification) ไปยัง Admin หรือแผนกที่เกี่ยวข้องเพื่อดำเนินการจัดส่งของรางวัลต่อไป

3.4 แดชบอร์ดสำหรับผู้ดูแลระบบ (Admin Dashboard)

ภาพรวมข้อมูล (Analytics Dashboard):

แสดงผลข้อมูลภาพรวมในรูปแบบกราฟและชาร์ตที่เข้าใจง่าย

Leaderboard: จัดอันดับพนักงานที่ได้รับเหรียญมากที่สุด (สามารถกรองตามรายสัปดาห์, รายเดือน, รายปี)

แสดงข้อมูลสถิติการให้-รับเหรียญ, อัตราการมีส่วนร่วมของพนักงาน

การตั้งค่าระบบ (System Configuration):

Admin สามารถกำหนดจำนวนเหรียญที่จะจัดสรรให้พนักงานในแต่ละเดือนได้

สามารถเปิด/ปิด หรือแก้ไขประเภทของเหรียญได้ (หากในอนาคตต้องการมีเหรียญหลายประเภท เช่น เหรียญขอบคุณ, เหรียญความคิดสร้างสรรค์)

การจัดการของรางวัล (Reward Management):

Admin สามารถเพิ่ม, ลบ, หรือแก้ไขรายการของรางวัลและจำนวน Point ที่ต้องใช้ได้

การจัดการผู้ใช้งาน (User Management):

Admin สามารถเพิ่มหรือลบข้อมูลพนักงานออกจากระบบได้"
